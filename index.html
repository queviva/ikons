that scene is composed of a series of ikons - svg images that can be instanciated and styled with css, or simply used as the src in an img tag

take a look at Rex - here he is as just an image tag, in his default state: a little bit of shade, a very faint logo on his prescription sweater, a white hat; he's just an image, nothing happens if you mouse-over him, or anything
now let's look at people like the objects they are! here's Poppy as the data attribute of an object tag - now if you mouseover him, you'll see a list of css-variables that can be used to customize him when he's finally instanciated in his most useful carnation
at last, we have Rex here as an svg use statement that has been customized for our own pleasure; when used this way, it's possible to have ikons dynamically match the page's themes

···
the actual best way to include svg is: as a use statement ikon; this combines the "ease of maintenance" of an external file with the css maniplability of inline svg - like this, in a page: 

            
<html>

<svg viewBox="0 0 100 100">
    <use xlink:href="rex.svg#ikon" />
</svg>

</html>
in the .svg file itself, put the ikon in a def tag, use css-var, fallbacks for default styling, and then create a use reference to the ikon, with a descriptive title 

            
<svg>

    <defs>
        <g id="ikon">
        
            <!--
                this rect will use the css
                variable '--rect-fill',
                otherwise it will fallback
                to the hard-coded color
            -->
            
            <rect fill="var(--rect-fill, #c30)"/>
            
        </g>
    </defs>
    
    <!--
        this separate group is what will be
        displayed if this file is used in
        an image, or object, tag
    -->
    
    <g>
    
        <!--
            explain which id's to use
            and what vars can be set
            in this title area
        -->
        
        <title>
        
            #ikon ::: [the id to use]
        
            --rect-fill [all the vars]
            ...
        
        </title>
        
        <!--
            instanciate the ikon here
            in a use tag, exactly as would
            be done inline on a page
        -->
        
        <use xlink:href="#ikon" />
        
    </g>
    

</svg>

when this type of file is xlink:ed in a use tag, the #ikon in the url refers to the def version, which attempts to find the css values before falling back on the hard times of default

<use xlink:href="file.svg#ikon" />

any of the default values can be left alone, or overwritten if the page includes those variable names

for example, both the Rex and the Poppy files have css vars called ‑‑key-light, ‑‑fill-light, and ‑‑back-light; this way, the page itself creates and manipulates css variables, overwriting them in both files, and creating the lighting effects of the scene - Chalkie here is also experiencing the same bottle of lightening
